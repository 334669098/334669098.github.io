Build a Documentation Website with Hugo and GitHub Pages

Introduction

Hugo is one of the most popular open-source static site generators. With its amazing speed and flexibility, Hugo makes building websites fun again.

Prerequisites

  1. You have a GitHub account.
  2. You have created a repository username**.github.io** in your GitHub account. In this document, 334669098.github.io is used as an example.
  3. Your GitHub Pages site is currently being built from the master branch. Learn more.

Procedure

Step 1: Install Hugo

  1. Download Hugo. In this document, Hugo for Windows 64-bit is used as an example. For other releases, see https://github.com/gohugoio/hugo/releases.

  2. Unzip the downloaded package to any path and add the path to the environment variable Path.

  3. Check the Hugo version. If the version number is displayed, the installation is complete.

    hugo -version

Step 2: Create a New Site

  1. Create a site directory. In this document, D:\Hugo\Sites is used as an example.

  2. Create a site in D:\Hugo\Sites.

    hugo new site hugodemohgz # `hugodemohgz` is an example site name.

Step 3: Add a Theme

  1. In the hugodemohgz directory, download the theme Techdoc.

    `git clone https://github.com/thingsym/hugo-theme-techdoc.git /themes/techdoc `  #`https://github.com/thingsym/hugo-theme-techdoc.git` is the URL of the theme in GitHub, and `/themes/techdoc` is the local directory of the downloaded theme.

    See https://themes.gohugo.io for more themes.

  2. Open the config.toml file, add a line theme = "techdoc" at the end, save and exit.

Step 4: Add Some Content

  1. Create an index document _index.md in the hugodemohgz\content directory.

    hugo new _index.md

  2. Edit _index.md, add content and save.

Step 5: Publish to GitHub Pages

  1. Build the site in the hugodemohgz\public directory.

    hugo

  2. Go to the hugodemohgz/public/ directory and initialize the Git repository.

    cd ~/hugodemohgz/public/  # Generated HTML files are stored in the `public` folder, which will be converted to a Git repository.
git init  # Initialize the Git repository.

  3. In the hugodemohgz/public/ directory, add a remote repository for the local repository.

git remote add origin git@github.com:334669098/334669098.github.io.git  # `334669098` is an example GitHub username.

  1. In the hugodemohgz/public/ directory, add a remote repository for the local repository.

    git remote add origin git@github.com:334669098/334669098.github.io.git  # `334669098` is an example GitHub username.
